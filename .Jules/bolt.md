## 2024-05-18 - Lazy-loading Images Strategy\n**Learning:** The application uses many direct `<img />` tags (e.g., in maps/grids/sections) that render below the fold without lazy loading. While components like `WebPImage.astro` provide optimized fallbacks, they are not universally applied. Applying `loading="lazy" decoding="async"` to native images in static sections is a low-risk, high-impact technique to optimize the initial bundle payload and LCP metrics without introducing architectural changes.\n**Action:** Use native `loading="lazy"` paired with `decoding="async"` on UI and list items that render offscreen initially.
